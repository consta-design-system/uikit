## Типы сообщений

За каждое сообщение отвечает свой компонент. Заголовок и описание [можно поменять](#текст),
изображение определяется компонентом.

### Нет доступа (ошибка 403)

Это сообщение можно показать пользователю, если возникает ошибка с HTTP-кодом 403:
у пользователя нет доступа к документам, расположенным по этому адресу.

Здесь можно подсказать пользователю, как получить доступ. Если доступ получить нельзя, лучше тоже сказать об этом.

<ExampleResponses403 />

```tsx
<Responses403 />
```

### Страница не существует (ошибка 404)

Это сообщение можно показать пользователю, если возникает ошибка с HTTP-кодом 403:
на сервере не существует страницы с таким адресом.

Здесь можно разместить ссылки на полезные страницы: обычно это главная страница сайта
и самые популярные разделы.

<ExampleResponses404 />

```tsx
<Responses404 />
```

### Ошибка сервера (500 ошибка)

Это сообщение можно показать пользователю, если возникает ошибка с HTTP-кодом 500:
это значит, что сервер не может обработать запрос, при этом точно не известно,
что случилось и когда всё заработает.

К сожалению, здесь нельзя сказать ничего конкретного (и это раздражает пользователей).
Можно только подсказать, что проблемы на сайте, а не у пользователя в системе, и посоветовать зайти чуть позже.

<ExampleResponses500 />

```tsx
<Responses500 />
```

### Сервис недоступен (503 ошибка)

Это сообщение можно показать пользователю, если возникает ошибка с HTTP-кодом 503:
он не может связаться с сервером (хотя с самим сервером всё в порядке).

Здесь можно подсказать, что проблема временная. Если вы проводите плановые технические работы,
во время которых сервис недоступен, стоит сказать об этом тоже.

<ExampleResponses503 />

```tsx
<Responses503 />
```

### Проблемы с интернетом

Это сообщение на случай, если у пользователя пропал доступ к сети.

Здесь можно посоветовать проверить соединение и качество связи.

<ExampleResponsesConnectionError />

```tsx
<ResponsesConnectionError />
```

### Нет данных (пустая коробка)

Это сообщение подходит, когда на странице должна быть какая-то информация, но ее еще нет.
Например, список покупок, которые пользователь еще не совершал.

Здесь стоит сказать, что всё в порядке и подсказать, что делать, чтобы данные появились.
Например, добавить кнопку «Перейти к покупкам».

Если вам не подходит это изображение, можно использовать [вариант с пустыми карманами](#нет-данных-пустые-карманы).

<ExampleResponsesEmptyBox />

```tsx
<ResponsesEmptyBox />
```

### Нет данных (пустые карманы)

Это сообщение для тех же случаев, что и [вариант с пустой коробкой](#нет-данных-пустая-коробка).
Подходит для случаев, когда на странице должна быть какая-то информация, но ее еще нет.
Например, список заметок, которые пользователь еще не добавил.

Здесь стоит сказать, что всё в порядке и подсказать, что делать, чтобы данные появились.
Например, добавить кнопку «Написать заметку».

<ExampleResponsesEmptyPockets />

```tsx
<ResponsesEmptyPockets />
```

Это сообщение говорит о том, что какой-то объект успешно удален. Подходит для случаев, когда
пользователь что-то удаляет, и вам нужно сообщить ему, что всё получилось.

Здесь стоит подсказать, что именно удалилось, и дать возможность вернуть всё обратно,
если пользователь удалил это по ошибке.

<ExampleResponsesDeleted />


Это сообщение о результате поиска: пользователь что-то искал, но найти это не получилось.

Здесь стоит подсказать, что сделать, чтобы получить результат: поменять данные в фильтрах,
уточнить запрос, подобрать другие слова.

<ExampleResponsesNothingFound />

```tsx
<ResponsesNothingFound />
```

### Успешный результат

Это сообщение об успешном результате процесса: пользователь отправляет письмо, платит за покупки,
создает заметку, добавляет сотрудника в список.

Здесь стоит четко написать, что произошло: письмо отправлено, платеж прошел, заметка сохранилась, сотрудник добавлен список.
Чтобы у пользователя не было сомнений, что всё в порядке.

```tsx
<ResponsesSuccess />
```

Здесь стоит чётко написать, откуда человек вышел, и дать возможность войти обратно.

## Текст

Вы можете поменять тексты под любым изображением. За заголовок отвечает свойство `title`, за описание под ним — `description`.

В заголовке точка не нужна.

В описании точку можно ставить, но не обязательно. Есть несколько вариантов:

- ставить точку всегда,
- не ставить никогда,
- если одно предложение в описании, точку не ставить, если несколько предложений — ставить.

Все варианты выглядят хорошо, главное быть последовательным и везде делать одинаково.

<ExampleResponsesText />

```tsx
<ResponsesSuccess
  size="m"
  title="Всё готово"
  description="Можно двигаться дальше: сначала два шага направо, потом четыре налево, и всё — только вперёд!"
/>
```

## Кнопки

За кнопки под сообщением отвечает свойство `actions`. Можно добавить одну или несколько.

<ExampleResponsesActions />

```tsx
/>
```

## Свойства

| Свойство                 | Тип               | По умолчанию | Описание                                       |
| ------------------------ | ----------------- | ------------ | ---------------------------------------------- |
| [`title?`](#текст)       | `React.ReactNode` | -            | Заголовок сообщения                            |
| [`description?`](#текст) | `React.ReactNode` | -            | Описание сообщения                             |
| [`size?`](#размер)       | `'m'`, `'l'`      | `'l'`        | Размер всех элементов                          |
| [`actions?`](#кнопки)    | `React.ReactNode` | -            | Кнопки под сообщением                          |
| `className?`             | `string`          | -            | Дополнительные CSS-классы для этого компонента |
