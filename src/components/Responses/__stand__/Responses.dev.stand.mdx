## Обзор

Компоненты `Responses` предназначены для отображения сообщений о различных состояниях приложения, таких как ошибки HTTP, отсутствие данных, успешные операции или выход из системы. Каждый тип сообщения представлен отдельным компонентом с уникальным изображением и возможностью настройки текста, кнопок и размера.

## Импорт

```tsx
import { Responses403 } from '@consta/uikit/Responses403';
import { Responses404 } from '@consta/uikit/Responses404';
import { Responses500 } from '@consta/uikit/Responses500';
import { Responses503 } from '@consta/uikit/Responses503';
import { ResponsesConnectionError } from '@consta/uikit/ResponsesConnectionError';
import { ResponsesDeleted } from '@consta/uikit/ResponsesDeleted';
import { ResponsesEmptyBox } from '@consta/uikit/ResponsesEmptyBox';
import { ResponsesEmptyPockets } from '@consta/uikit/ResponsesEmptyPockets';
import { ResponsesNothingFound } from '@consta/uikit/ResponsesNothingFound';
import { ResponsesSuccess } from '@consta/uikit/ResponsesSuccess';
import { ResponsesExit } from '@consta/uikit/ResponsesExit';
```

## Свойства

| Свойство                                | Тип               | По умолчанию | Описание                                       |
| --------------------------------------- | ----------------- | ------------ | ---------------------------------------------- |
| [`title?`](#заголовок-и-описание)       | `React.ReactNode` | -            | Заголовок сообщения                            |
| [`description?`](#заголовок-и-описание) | `React.ReactNode` | -            | Описание сообщения                             |
| [`actions?`](#кнопки)                   | `React.ReactNode` | -            | Кнопки под сообщением                          |
| [`size?`](#размер)                      | `'m' \| 'l'`      | `'l'`        | Размер всех элементов                          |
| `className?`                            | `string`          | -            | Дополнительные CSS-классы для этого компонента |

## Содержимое

### Заголовок и описание

Свойства `title` и `description` позволяют настроить текст сообщения. `title` задаёт заголовок, а `description` — текст описания под ним.

- В заголовке точку ставить не нужно.
- В описании точку можно ставить или не ставить. Рекомендуется придерживаться одного стиля:
  - Всегда ставить точку.
  - Никогда не ставить точку.
  - Ставить точку, если в описании несколько предложений, и не ставить, если предложение одно.

  title="Всё готово"
  description="Можно двигаться дальше: сначала два шага направо, потом четыре налево, и всё — только вперёд!"
/>

### Кнопки

Свойство `actions` позволяет добавить одну или несколько кнопок под сообщением. Это полезно для предоставления пользователю дальнейших действий, например, перехода на другую страницу или отмены операции.

<MdxTabs>

<ExampleResponsesActions />

```tsx
<ResponsesSuccess
  size="m"
  title="Всё готово"
  description="Можно двигаться дальше: сначала два шага направо, потом четыре налево, и всё — только вперёд!"
  actions={<Button size="m" view="ghost" label="Ура, я уже бегу" />}
/>
```

</MdxTabs>

## Внешний вид

### Размер

Свойство `size` определяет размер всех элементов компонента. Доступные значения: `'m'` и `'l'`. По умолчанию используется `'l'`.

```tsx
<Responses404 size="m" />
```

<ExampleResponsesSizeM />

```tsx
<Responses404 size="l" />
```

<ExampleResponsesSizeL />

### Адаптивность

Компоненты `Responses` автоматически заполняют всё пространство родительского контейнера и центрируются в нём. Например, в контейнере шириной 300 px компонент `ResponsesExit` будет выглядеть следующим образом:

```tsx
<ResponsesExit />
```

<ExampleResponsesAdaptive />

## Типы сообщений

Каждый тип сообщения реализован отдельным компонентом с уникальным изображением, которое нельзя изменить. Текст и кнопки можно настраивать через свойства `title`, `description` и `actions`.

### Нет доступа (ошибка 403)

Используется для HTTP-ошибки 403, когда у пользователя нет доступа к ресурсу. Рекомендуется подсказать, как получить доступ, или указать, что доступ невозможен.

```tsx
<Responses403 />
```

<ExampleResponses403 />

### Страница не существует (ошибка 404)

Показывается при HTTP-ошибке 404, когда страница не найдена. Полезно добавить ссылки на главную страницу или популярные разделы.

```tsx
<Responses404 />
```

<ExampleResponses404 />

### Ошибка сервера (500 ошибка)

Применяется для HTTP-ошибки 500, когда сервер не может обработать запрос. Рекомендуется сообщить, что проблема на стороне сервера, и посоветовать вернуться позже.

```tsx
<Responses500 />
```

<ExampleResponses500 />

### Сервис недоступен (503 ошибка)

Используется при HTTP-ошибке 503, когда сервер временно недоступен. Укажите, что проблема временная, и, при необходимости, упомяните плановые технические работы.

```tsx
<Responses503 />
```

<ExampleResponses503 />

### Проблемы с интернетом

Сообщение для случаев, когда у пользователя пропал доступ к сети. Рекомендуется посоветовать проверить соединение.

```tsx
<ResponsesConnectionError />
```

<ExampleResponsesConnectionError />

### Нет данных (пустая коробка)

Подходит для отображения отсутствия данных, например, пустого списка покупок. Рекомендуется подсказать, как добавить данные, например, через кнопку.

```tsx
<ResponsesEmptyBox />
```

<ExampleResponsesEmptyBox />

### Нет данных (пустые карманы)

Альтернатива сообщению «Пустая коробка» для тех же случаев. Например, для пустого списка заметок. Укажите, как добавить данные.

```tsx
<ResponsesEmptyPockets />
```

<ExampleResponsesEmptyPockets />

### Информация удалена

Сообщение для подтверждения успешного удаления объекта. Рекомендуется указать, что было удалено, и предложить возможность восстановления.

```tsx
<ResponsesDeleted />
```

<ExampleResponsesDeleted />

### Ничего не нашлось

Используется, когда поиск не дал результатов. Подскажите, как уточнить запрос или изменить фильтры.

```tsx
<ResponsesNothingFound />
```

<ExampleResponsesNothingFound />

### Успешный результат

Сообщение об успешном выполнении действия, например, отправке письма или добавлении сотрудника. Чётко укажите, что произошло.

```tsx
<ResponsesSuccess />
```

<ExampleResponsesSuccess />

### Результат выхода

Сообщение об успешном выходе из системы, например, из личного кабинета. Укажите, откуда выполнен выход, и предложите возможность входа обратно.

```tsx
<ResponsesExit />
```

<ExampleResponsesExit />
