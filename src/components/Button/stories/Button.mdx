# Button

Кнопка — один из самых популярных элементов интерфейса. Компонент Кнопка обладает богатым набором параметров, сочетания которых покрывают большинство основных сценариев использования.

## Когда использовать

В современных веб-интерфейсах кнопка — один из главных элементов. Кнопка может запускать как действие на текущей страницы, так и переход на другую страницу (функция гиперссылки). Разнообразие `цветов`, `форм`, `размера` и других параметров позволяет использовать кнопки для самых разных кейсов и сценариев.

Любую кнопку можно выбрать с клавиатуры с помощью табуляции. Когда кнопка попадает в фокус, вокруг неё появляется рамка. В этом состоянии на кнопку можно нажать с помощью клавиши `Enter`.

## Название

Кнопки используются для обозначения и выбора действий. Например, «Добавить документ», «Удалить запись», «Оплатить налог» и других.

> ✅ Название кнопки пишется с заглавной буквы, не используйте верхний регистр

Название кнопки должно однозначно говорить о том, что произойдет если на нее нажать.

<ButtonExampleNameSemantics />

Для названий кнопок используется совершенный вид инфинитива: «Сохранить», «Скачать отчет», «Найти». Название должно отвечать на вопрос «Что сделать?»

<ButtonExampleNameVerb />

Не называйте кнопку в две строки: делайте кнопку шире или меняйте название.

Нельзя сокращать многоточием название кнопки, если оно не входит в целевое разрешение экрана. Сокращение названия допускается при адаптиве, но рекомендуется использовать подходящие формулировки для всех используемых разрешений экрана.

## Внешний вид

Кнопка имеет 4 варианта внешнего вида: `primary`, `secondary`, `ghost`, `clear`. Такой набор помогает правильно распределить акцент пользовательских действий на странице. Отвечает за внешний вид параметр `view`.

### Primary

Кнопка призыва к действию (CTA – англ. call to action), в зависимости от ситуации, обычно побуждает пользователей зарегистрироваться/приступить к следующему шагу/залогиниться и т.д. Такие кнопки нужно использовать для самых важных действий, на которые нужно сделать наибольший акцент.

<ButtonExampleViewPrimary />

### Secondary

Secondary кнопка – это как кнопка Primary, только с чуть меньшим акцентом. Используется в ситуациях, когда мы хотим сохранить важность действия, но не слишком привлекать к этому внимание.

<ButtonExampleViewSecondary />

### Ghost

Ghost кнопка – это кнопка «Назад» возле первичной кнопки «Далее». Или кнопка «Отмена» возле кнопки «Подтвердить». Ghost кнопки – это альтернатива Primary действия, которое мы предоставляем пользователям, а также просто действия, на котором мы не хотим делать сильный акцент.

<ButtonExampleViewGhost />

### Clear

Тип Clear имеет прозрачный фон, но ведет себя как полноценная кнопка. Её нужно использовать, когда внимание пользователя должно быть сосредоточено на более важных контролах. Также кнопки Clear идеально подходят для списка иконочных кнопок третьестепенного действия.

<ButtonExampleViewClear />

### Disabled

Все четыре внешних вида имеют одинаковый параметр `disabled`, который делает кнопки одинаково серыми и блокирует интерактивность кнопки в тех случаях, когда пользователь не может его выполнить в силу каких-то условий. 

<ButtonExampleViewDisabled />


## Размеры

Параметр `size` отвечает за размер кнопок. Сейчас есть 4 варианта размерности: `l`, `m`, `s`, `xs`. Ширина кнопки задается размером контента внутри нее и стандартными отступами. 

<ButtonExampleSizeBasic />

Для каждого из четырёх размеров доступна ширина кнопки в 100% ширины родительского контейнера.

<ButtonExampleSizeFull />


### Размер иконки

По умолчанию размер иконки зависит от размера кнопки.

|   |   |   |   |   |
| - | - | - | - | - |
| Размер кнопки | L | M | S  | XS |
| Размер иконки | M | S | XS | XS |

В кнопках могут использоваться иконки разных размеров, в зависимости от ситуации. За это отвечает параметр `iconSize`, в котором можно указать любой размер иконки из списка допустимых: `m`, `s`, `xs` 


## Кнопка с иконкой

Название кнопки можно дополнять иконкой. Она поможет быстрее найти нужную кнопку в ряду других и быстрее понять, что делает кнопка.

Для каждого из четырех размеров кнопки доступны две модификации: `iconLeft` слева и `iconRight` справа от текста. Оба эти параметра ожидают компонент конкретной иконки из нашей библиотеки [иконок](?path=/docsx/icons--icons).

<ButtonExampleIconBasic />

> ❗️ В панели с большим количеством кнопок, иконки будут создавать лишний визуальный шум. В этом случае оставьте иконки только у самых важных действий

Кнопка может состоять только из иконки. Такие кнопки экономят место и позволяют разместить рядом много похожих кнопок. Например кнопки выбора даты и фильтрации. Важно помнить, что метафора иконки должна быть пользователю знакома или легко считываться сразу, иначе пользователь не поймет что делает кнопка.

<ButtonExampleIconOnly />

Для создания такой кнопки надо использовать булевый параметр `onlyIcon` в паре с любым из `iconLeft` или `iconRight`


## Форма

За форму кнопки отвечает параметр `form`. Помимо варианта по умолчанию `default`, кнопки могут быть круглыми `round` и квадратными `brick`.

Для комбинации Кнопки с [Полем ввода](?path=/story/textfield--%D1%82%D0%B5%D0%BA%D1%81%D1%82%D0%BE%D0%B2%D0%BE%D0%B5-%D0%BF%D0%BE%D0%BB%D0%B5-%D0%B2%D0%B2%D0%BE%D0%B4%D0%B0) предусмотрены гибридные формы. У таких вариантов правая и левая грани могут отличаться. В примере ниже параметр `form` у Кнопки равен `brick-round`.

## Загрузка

В ситуациях, когда после нажатия на кнопку пользователь вынужден дождаться какого-то результата, в Кнопке можно отобразить лоадер загрузки. Для этого надо указать булевый параметр `loading`.
