# Дизайн-система Consta

Дизайн-система «[Газпром нефти](https://www.gazprom-neft.ru/)». Распространяется бесплатно по лицензии MIT, подходит для создания цифровых продуктов: веб-сервисов и не только.

- [Сайт дизайн-системы](http://consta.gazprom-neft.ru)
- [Телеграм-канал](https://t.me/consta_ui_releases) с новостями и релизами

## Что входит в дизайн-систему

Несколько библиотек с интерфейсными компонентами
и правилами их взаимодействия. Компоненты — кнопки, иконки, списки, таблицы и другие элементы, из которых собирается интерфейс, реализованы в двух форматах: для дизайнеров, чтобы собирать макеты в Figma, и для разработчиков — в виде кода на React.

[Библиотеки для разработки](?path=/docs/common-about-github--page)

## Технологии

Consta построена по принципам [БЭМ-методологии](https://ru.bem.info/).
В основе дизайн-системы — библиотека [whitepaper](https://github.com/whitepapertools).

Сейчас библиотека компонентов дизайн-системы существует только в React-реализации. Что мы используем:

- [React](https://reactjs.org/)
- [Typescript](https://www.typescriptlang.org/)
- PostCSS ([postcss-nested](https://github.com/postcss/postcss-nested), [postcss-color-function](https://github.com/postcss/postcss-color-function), [postcss-simple-vars](https://github.com/postcss/postcss-simple-vars) для media-query)
- [whitepaper](https://github.com/whitepapertools)

Библиотека рассчитана на браузеры семейства Chromium.

> Мы не поддерживаем Internet Explorer и старую версию Edge, компоненты в этих браузерах могут отображаться некорректно

Если нужен fallback для IE, есть два решения:

1. Форк от библиотеки с фоллбэками.
2. Собственная реализация по методологии дизайн-системы.

## Особенности

#### Гибкая тематизация

Тема определяет внешний вид всего интерфейса — цвета, шрифты, отступы. В тему вкладываются все компоненты, а ещё темы вкладываются друг в друга. В дизайн-системе
может быть сколько угодно вариантов темы. Поэтому:

- В любой момент можно поменять внешний вид всего интерфейса, в том числе – по желанию пользователя (например, включить тёмную цветовую схему по кнопке или когда стемнеет).
- Можно поменять внешний вид не всего интерфейса, а какой-то его части, например, бокового меню или одной карточки в списке.
- Легко создать свой вариант оформления и применить ко всему проекту, например, перекрасить приложение в цвета бренда.

[Подробнее о темах](?path=/docs/thematization-what-are-themes--page)

#### Семантические переменные

Все параметры темы описаны с помощью переменных ([CSS Custom Properties](https://developer.mozilla.org/ru/docs/Web/CSS/Using_CSS_custom_properties))&nbsp;— значения цветов, отступов и размеров типографики берутся из темы, а значит, легко меняются
вместе с ней.

Переменные названы семантически и используются по смыслу — по названию переменной
несложно понять, зачем она нужна. Например, переменная `--color-typo-link` не должна использоваться для фона блока, а переменная `--color-bg-warning` используется только для отображения статуса, а не просто для раскрашивания фона в оранжевый.

#### Кастомные блоки и компоненты

В дизайн-системе есть готовые блоки, но из её компонентов можно собирать кастомные блоки и элементы интерфейса. Если чего-то не хватает, можно создавать своё
по принципам дизайн-системы.

[Как добавить новый компонент](?path=/docs/common-addcomponent--page)
